## Unreleased

- **Cancel claim — post-collection ship-back path.** Cancelling once the device is already with Revibe no longer reverts to Delivered (which wrongly implied no pickup happened); it now routes to a pay-return-shipping flow to get the device back. `canCancelClaim` widened to the whole pre-completion window (`initiated`/`pickup`/`qc`); new `cancelNeedsShipBack(claim)` picks the path — clean revert for pre-pickup device claims + all compensation, ship-back for device claims at `pickup`/`qc`. Journeys: new `claim_cancelled_shipback` node on the three device journeys, wired into `claim_qc_started` + `claim_reset_failed`, pointing into the existing `claim_return_shipping_paid` return chain. So the `ResetFailedCard` (`qc`) cancel now correctly pays-to-return rather than reverting. `src/lib/claims.js`, `src/App.jsx`, `CancelClaimSheet.jsx`, `InvalidClaimCard.jsx`, `src/data/journeys/claim{ChangeOfMind,Issue,Warranty}.js`.
