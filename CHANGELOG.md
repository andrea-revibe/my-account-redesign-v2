Internal demo project. One line per material change, newest at the top. Phase headings preserved for git/PR cross-reference; for the full "why" check the corresponding commit message or the per-feature doc in `docs/output/`.

## Unreleased

- **`InvalidClaimCard`'s `Pay` / `Decline` / `I changed my mind and will pay` CTAs now drive the journey (dev panel moves in lockstep), and the card's state is derived from the order data instead of a local `mode` + sync effect.** Previously the buttons only toggled local card state while the dev panel stayed at the `claim_invalid_confirmed` step, and the reverse-after-decline silently desynced (the panel advanced but the card stayed on the closed state, because the order ended up with both `declinedAt` and `paidAt` and the forward-only effect only flipped `paid` *from* `action_needed`). Now: `Pay` and the reverse both call a `validNext`-gated `handlePayReturnShipping` (`App.jsx`) → `claim_return_shipping_paid`; `Decline` calls `handleDeclineReturn` → `claim_invalid_declined`. `claim_invalid_declined` is **no longer terminal** — it forks to `claim_return_shipping_paid`, so the reverse advances forward in lockstep (added in all three journeys: `claimChangeOfMind`, `claimIssue`, `claimWarranty`). `InvalidClaimCard` now derives `mode` from `invalidClaim.paidAt`/`declinedAt` (**paidAt wins**, so the lingering `declinedAt` after a reversal still reads as paid) with a `localMode` override for the standalone mock; the forward-only sync effect is gone, so a dev-panel **Back** re-syncs the card both ways. No notification copy touched (`claim.ship_back.created` / `claim.declined` already authored). `src/components/InvalidClaimCard.jsx`, `src/App.jsx`, `src/data/journeys/{claimChangeOfMind,claimIssue,claimWarranty}.js`. See `docs/output/returns/claim_tracking.md` §3.3, `journey_backend_spec.md`.

New event `claim.reset.completed` is unauthored ⇒ silent (no notification copy touched). `src/components/ResetFailedCard.jsx`, `src/App.jsx`, `src/data/journeys/{claimChangeOfMind,claimIssue,claimWarranty}.js`.
